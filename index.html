Как это работает ?


var file = new java.io.File(path);
  • Виртуальная машина выполняет JavaScript код
  • Биндингом определяются соответствующие методы вызова с нативной стороны
  • "Сервис передачи" преобразует строку JavaScript в объект java.lang.String
  • Результатом вызова функции будет кастомный объект JavaScript, служащий в качестве прокси для нативного объекта java.io.File, созданного на стороне Андроида. Когда метод вызван для этого прокси, он делегируется лежащему в основе инстансу нативного объекта.

Как это выглядит в коде ?

Пример из документации:

// iOS
var array = new NSMutableArray();
array.addObject(new NSObject());

// Android
var context = ...;
var button = new android.widget.Button(context);
// "My Button" is converted to java.lang.String
button.setText("My Button");

Как это выглядит в коде ?

Пример из жизни:

var listView = this.page.getViewById("list_view");
listView.on(ListView.itemLoadingEvent, (args: ItemEventData)=> {
    // disable select colors if it iOS

    }
});

QA: Отзывы

  • - Ломается практически любая кнопка, если успеть нажать на "back" на Андроид-девайсе в момент анимации.
  • + Список из 3000 объектов резво крутится ;-)

Разработчики: Трудности

  • CLI: Не всегда с первого раза запускается эмулятор.
  • При составлении структуры компонента из стандартных шаблонов разметки периодически выбрасывает из эмулятора, либо просто выдаёт белый экран. Очень тяжело определить причину такого поведения.

Разработчики: Трудности

  • Нет стабильного инспектора для livesync, нет возможности смотреть разметку.
  • Нет возможности использования стандартных html-элементов разметки. Стилизация готовых шаблонов затрудняется невозможностью увидеть их базовые CSS-свойства. Соответственно, иногда не получается перебить некоторые свойства.

Разработчики: Понравилось

  • CLI.
  • Наличие готовых анимаций.
  • Наличие готовых универсальных элементов разметки, которые по разному отрисовываются в зависимости от платформы (т.е., один код для всех платформ).

Разработчики: Понравилось

  • Довольно удобные layout containers. Можно быстро "набросать" простой дизайн.
  • Есть возможность разработки сразу под две платформы – разделение и автоматический выбор файлов (темплейтов, CSS) .android и .ios, в зависимости от билда.
  • Связка Angular 2 + RxJS как гибкое средство для асинхронных операций.

Разработчики: Не понравилось

  • Менее удобно, чем веб-разработка.
  • Затруднена стилизация при наличии собственного дизайна.
  • Появление ошибок при нестыковке некоторых шаблонов разметки между собой.
  • Для быстрой и качественной разработки необходимо знать тонкости Angular 2 и нативной платформы.
